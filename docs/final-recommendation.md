# 终端复制功能 - 最终建议

## ⚠️ 问题本质

tmux 在 copy-mode 中执行任何改变选择状态的操作时，**会自动调整视图将光标位置显示在可见区域**。这是 tmux 的核心设计行为，无法通过配置完全禁用。

**具体表现**：
- 复制文本后，视图会滚动到复制的行
- 清除选择后，视图会调整到光标位置
- 这是 tmux 的固有限制，不是 bug

---

## 🎯 三种可行方案

### 方案1：接受滚动（推荐继续使用）

**配置**：保持当前配置

**优点**：
- ✅ 复制功能完整
- ✅ 支持 vim/top 等 alternate screen
- ✅ 支持滚轮翻看历史

**缺点**：
- ⚠️ 复制后视图会调整（tmux 固有行为）

**适用场景**：
- 用户对轻微滚动不敏感
- 需要完整的 tmux 功能

---

### 方案2：Shift+鼠标拖选（无滚动）

**操作步骤**：
```
1. 按住 Shift 键
2. 鼠标拖选文本
3. 按 Cmd+C (Mac) 或 Ctrl+C (Win/Linux)
4. ✅ 完成
```

**优点**：
- ✅ **无滚动**
- ✅ **无抖动**
- ✅ 支持 Cmd+C (Mac)
- ✅ 简单直观

**缺点**：
- ❌ 无法在 vim/top 等 alternate screen 中使用
- ❌ 无法使用滚轮翻看历史

**适用场景**：
- 主要在命令行模式下工作
- 对滚动敏感，需要流畅体验

---

### 方案3：禁用 mouse mode（键盘党）

**配置**：
```bash
set-option -g mouse off
```

**操作步骤**：
```
1. Ctrl+B [  （进入 copy-mode）
2. 空格键    （开始选择）
3. 方向键    （调整范围）
4. Ctrl+C    （复制）
5. q          （退出）
```

**优点**：
- ✅ 无鼠标相关滚动问题
- ✅ 纯键盘操作效率高

**缺点**：
- ❌ 无法使用鼠标
- ❌ 无法使用滚轮翻看历史

**适用场景**：
- 键盘操作熟练的用户
- 服务器运维场景

---

## 📊 对比总结

| 特性 | 方案1: 接受滚动 | 方案2: Shift+拖选 | 方案3: 禁用 mouse |
|------|---------------|------------------|------------------|
| 滚动问题 | ⚠️ 有 | ✅ 无 | ✅ 无 |
| 抖动问题 | ⚠️ 轻微 | ✅ 无 | ✅ 无 |
| vim/top 支持 | ✅ 支持 | ❌ 不支持 | ✅ 支持 |
| 滚轮翻历史 | ✅ 支持 | ❌ 不支持 | ❌ 不支持 |
| Mac Cmd+C | ❌ 不支持 | ✅ 支持 | ❌ 不支持 |
| 鼠标支持 | ✅ 支持 | ⚠️ 部分 | ❌ 不支持 |
| 学习成本 | 低 | 低 | 高 |

## 🎯 我的建议

### 对于你的使用场景

考虑到你：
- 对滚动敏感
- 使用 Mac
- 需要流畅的复制体验

**推荐使用方案2：Shift+鼠标拖选**

**日常操作流程**：
1. 普通命令行操作 → **Shift+鼠标拖选** → Cmd+C
2. vim/top 中 → 使用 vim 内置的 `yy` 等复制命令
3. 需要翻看历史 → 使用 `Ctrl+B [` 进入 copy-mode

---

## 🔧 快速切换方案

### 切换到方案2（Shift+拖选）

```bash
# 禁用 tmux mouse mode
docker exec wetty-mcp-terminal-wetty-mcp-1 tmux set-option -g mouse off
```

### 切换回方案1（完整功能）

```bash
# 启用 tmux mouse mode
docker exec wetty-mcp-terminal-wetty-mcp-1 tmux set-option -g mouse on
```

## ✅ 当前已实现的功能

无论选择哪个方案，以下功能都已正常工作：

1. ✅ **ESC 键**：只清除选择，不退出 copy-mode
2. ✅ **Ctrl+C**：复制 + 清除选择 + 不退出 copy-mode
3. ✅ **鼠标拖选**：自动进入 copy-mode
4. ✅ **松开鼠标**：保持选择状态
5. ✅ **OSC 52**：复制内容通过 WebSocket 推送到前端
6. ✅ **剪贴板集成**：自动复制到浏览器剪贴板
7. ✅ **Toast 提示**：复制成功后显示提示
